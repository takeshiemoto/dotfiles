## エージェント・ガイドライン

- 過剰（病的）なまでの正確さよりも、常にシンプルさを優先してください。
- YAGNI、KISS、DRYの原則に従うこと。
- 循環的複雑度を増やすことなく、実質コストゼロで実装できる場合を除き、後方互換性のためのシム（調整用コード）やフォールバック（代替）パスは含めないでください。
- `npx` の使用禁止。`npm run <script>` 経由で実行すること。
- 冗長なコメントを禁止する。コードで語る。
- 複数案を並べず、熟考したベスト案を1つだけ提案する。
